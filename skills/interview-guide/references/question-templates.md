# 面试问题设计模板

> 行为面试 + 情境面试的问题设计方法论。这是 interview-guide 生成问题的底层模板。

---

## 两大问题类型

### 行为面试（Behavioral）— "讲一个你过去…"

**原理**：过去的行为是预测未来行为的最佳指标。通过让候选人讲述**真实经历**，判断其能力真实性。

**结构**：用 **STAR** 引导回答：
- **S**ituation（情境）："当时的情况是什么？"
- **T**ask（任务）："你负责做什么？"
- **A**ction（行动）："你具体做了什么？怎么做的？"
- **R**esult（结果）："结果怎样？有数据吗？你学到了什么？"

**好题 vs 废题**：

```
❌ 废题："你做过Unity项目吗？"（是/否就答完了）
❌ 废题："请介绍一下你的项目经历"（太开放，候选人会背稿）
✅ 好题："你简历写'负责战斗系统的客户端开发'，
        能讲讲其中最复杂的一个战斗技能是怎么实现的吗？
        （追问）动画状态机你们怎么设计的？技能连招的逻辑怎么处理？"
```

### 情境面试（Situational）— "如果遇到…你会…"

**原理**：给一个**假设的工作场景**，看候选人的思路和判断力。考察的不是"答对"，是**思维方式**。

**结构**：
1. 给具体场景（有约束条件：时间/资源/冲突）
2. 问"你会怎么处理/设计/决策"
3. 追问"为什么这么选""还有什么方案""风险是什么"

**好题 vs 废题**：

```
❌ 废题："你觉得性能优化重要吗？"（谁都会说重要）
✅ 好题："线上Spine动画内存泄漏，帧率掉到15fps，给你2小时排查，
        你的思路是什么？先看什么？怎么定位是哪个动画的问题？"
```

## 各轮次的问题设计要点

### 一面（技术面）— 深度验证

**目标**：验证简历技术能力的真实性 + 深度。

**问题配方**：60% 行为面试（简历深挖）+ 40% 情境面试（技术场景）

**设计要点**：
1. **锚定简历原文**：问题必须引用候选人简历里的具体项目/技能表述
2. **追问到指标**：技术题要追到"数据/指标/方案选型"，不停在"你做过吗"
3. **设计分层问题**：先问"做过什么"（过滤造假），再问"怎么做"（验证深度），最后问"为什么这么选"（验证理解）

### 二面（业务负责人）— 综合判断

**目标**：团队适配 + 综合评估 + 验证一面疑点。

**问题配方**：70% 行为面试（协作/抗压/成长）+ 30% 情境面试（团队协作场景）

**设计要点**：
1. **不重复考技术**：技术一面考过了，二面考"技术之外的"
2. **重点问协作**："你和策划/美术/产品意见冲突时怎么处理"
3. **验证稳定性**：职业规划、离职原因、对加班/压力的态度
4. **深挖一面疑点**：如果一面评价里有担忧，二面要验证

### 三面（HR面）— 价值观 + 动机

**目标**：谦卑/渴求/聪慧 + 薪资/时间/背调。

**问题配方**：80% 行为面试 + 20% 动机确认

**设计要点**（按三大维度）：

**谦卑**：
- "讲一个你犯过的最大错误，后来怎么处理的？"
- "你觉得自己最大的不足是什么？在做什么改进？"
- "上次你的方案被否定时，你怎么反应的？"

**渴求**：
- "你未来3年的目标是什么？在做什么准备？"
- "最近主动学了什么新东西？怎么学的？"
- "你觉得自己在同龄人里排第几？为什么？"

**聪慧**：
- 现场给一个新问题（不需要答对，看思路和反应）
- 观察沟通中的逻辑性、倾听能力、理解速度
- "你对我们公司/这个岗位了解多少？"（看是否做了功课）

### 四面（终面）— 潜力判断

**目标**：系统思考 / 洞察力 / 追求极致 / 学习能力。

**问题配方**：40% 开放性大题 + 30% 行业洞察 + 30% 学习方法

**设计要点**：
1. **开放性大题**（看系统思考）：
   - "如果让你从0设计我们的XX系统，你会怎么拆？"
   - "给你3个月和2个人，你怎么做一个XX的MVP？"
2. **行业洞察**（看洞察力）：
   - "你觉得XX行业/技术未来3年会怎么变？为什么？"
   - "你觉得我们竞品里XX做得好的地方是什么？我们能不能学？"
3. **追求极致**：
   - "你做过最满意的作品/项目是什么？为什么？还能更好吗？"
   - "你怎么判断一个方案'够好了'vs'还可以优化'？"
4. **学习能力**：
   - "最近学了什么新技术/新工具？学习路径是什么？"

## 按岗位类型的问题差异

### 技术岗（开发/测试/运维）
- 行为题锚定：项目架构、性能指标、线上故障处理
- 情境题方向：系统设计、性能优化、技术选型
- 追问重点：方案选型的 Why、指标数据、踩坑经历

### 设计岗（UI/交互/美术/动效）
- 行为题锚定：作品集里的具体项目、设计决策
- 情境题方向：给设计需求让现场出思路、设计review场景
- 追问重点：设计风格的适配度、和策划/程序的协作方式、审美判断力

### 运营岗（内容/用户/社区）
- 行为题锚定：具体活动的数据、创作者/用户关系处理
- 情境题方向：给一个运营目标让制定策略、危机处理
- 追问重点：数据敏感度、内容判断力、用户同理心

### 产品岗（产品经理/数据PM）
- 行为题锚定：负责过的模块从需求到落地、数据驱动决策的经历
- 情境题方向：MVP范围定义、需求优先级排序、资源不足的取舍
- 追问重点：独立负责的真实性、数据分析能力、AI方向的理解深度

## 问题质量的 5 条红线

生成问题时，绝不突破：

1. **不生成"是/否"封闭题**——"你做过XX吗"改成"你做XX时遇到最复杂的情况是什么"
2. **不生成"介绍下你自己"开放题**——太泛，候选人会背准备好的稿
3. **不生成与简历无关的通用题**——所有行为题必须锚定简历原文
4. **不生成理论背诵题**——"什么是OOP"改成"你在XX项目里怎么用OOP解决的XX问题"
5. **每轮问题不超过 8 个**——面试时间有限，深度优先于广度
