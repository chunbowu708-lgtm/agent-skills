# 公司面试评分表完整维度

> 这是公司 5 张真实评分表的维度，interview-guide 生成问题时照此映射。
> 每张表列：评价维度 + 评分档位标准（档位帮助面试官打分，也帮助 AI 判断"什么是好答案"）。

---

## 1. 部门初试评价表（一面·技术面）

> 一面用此表。核心是验证专业能力——简历写的是不是真的、深度够不够。

### 评价维度

| 维度 | 考察什么 | 评分参考 |
|------|---------|---------|
| **教育背景及所接受的专业培训** | 基础是否扎实、专业是否对口 | 基本胜任 / 不足 |
| **原有经验或资源应用到新工作中情况** | 过往经验能不能迁移过来 | 较有潜力 / 基本胜任 / 不能胜任 |
| **对解决专业问题、案例的回答情况** | 真实技术水平（核心） | 较有潜力 / 基本胜任 / 不能胜任 |
| **能否满足新岗位对专业工具应用的要求** | 工具掌握深度 | 不能胜任 / 基本胜任 / 胜任 |
| **弥补所欠缺的经验需要时间长短** | 学习曲线（A=不需要/快速，D=需很长时间） | A / B / C / D |

### 综合评价 + 得分 + 结论 + 记录

面试官写综合评价文字 + 打分 + 结论（通过/不通过）+ 记录候选人基本信息。

---

## 2. 部门复试评价表（二面·业务负责人面）

> 二面用此表。核心是综合判断——这个人来了能不能干活、适不适合团队。

### 评价维度

| 维度 | 考察什么 |
|------|---------|
| **优点** | 候选人的核心优势（综合一面反馈深挖） |
| **缺点** | 核心短板（验证一面的担忧，排除"面霸"） |
| **综合评价** | 整体判断：团队适配、成长性、稳定性 |
| **建议职级** | 对标内部同岗位人员定级 |
| **结论** | 录用 / 不录用 / 待定 + 理由 |

> 二面不是重新考技术，是在一面基础上做**综合判断**。重点问：团队协作风格、抗压能力、职业规划稳定性。

---

## 3. 人力资源面试评价（三面·HR面）

> 三面用此表。核心是价值观和软素质——三个维度来自经典的人力资源评估框架。

### 评价维度（三大维度）

| 维度 | 考察什么 | 怎么问 |
|------|---------|--------|
| **谦卑（Humble）** | 能否承认不足、接受反馈、不傲慢 | "讲一个你犯过的最大错误""你觉得自己最大的不足是什么" |
| **渴求（Hungry）** | 对成长/成就的驱动力、自我要求 | "你未来3年的目标""最近在主动学什么" |
| **聪慧（Smart）** | 思维敏捷、沟通顺畅、人际判断力 | 现场给新问题看反应速度；观察沟通中的逻辑性 |

### 综合评价 + 结论 + 记录

HR 面不重复考技术，专注：动机稳定性、薪资期望、入职时间、背调风险。

## 4. 校招面试评分表 - 技术类（终面·技术岗）

> 校招/技术岗终面用此表。核心是潜力——技术深度 + 思维方式。

### 评价维度

| 维度 | 考察什么 | 评分参考 |
|------|---------|---------|
| **系统思考** | 能否看全局、做架构/方案判断（不是只会写代码） | 优秀 / 良好 / 一般 / 不足 |
| **洞察能力** | 能否看到问题本质、行业趋势 | 优秀 / 良好 / 一般 / 不足 |
| **追求极致** | 有没有高标准、死磕细节的意识 | 优秀 / 良好 / 一般 / 不足 |
| **学习能力** | 新事物吸收速度、学习方法 | 优秀 / 良好 / 一般 / 不足 |
