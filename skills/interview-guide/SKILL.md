# 面试考核维度问答

## 这个 skill 解决什么

面试官拿到候选人简历后，常面临两个问题：
1. **"面什么"**——不知道该重点考察哪些维度（尤其是跨部门的面试官，不熟悉这个岗位）
2. **"怎么问"**——问的都是"介绍下你的项目"这种废话，问不出真实水平

本 skill 照公司 **5 张真实评分表**的维度，针对候选人简历薄弱点，产出每轮面试的考察重点 + 具体问题（行为面试 + 情境面试都要有）。

## 公司面试评分表体系（4轮 × 5张表）

公司面试分 4 轮，每轮用不同的评分表（详见 `references/scoring-rubrics.md`）：

| 轮次 | 角色 | 用哪张评分表 | 核心考察 |
|------|------|------------|---------|
| **一面** | 技术面（技术负责人） | 部门初试评价表 | 专业能力：教育背景、经验应用、专业问题回答、工具掌握、弥补欠缺时间 |
| **二面** | 业务负责人面 | 部门复试评价表 | 综合判断：优点、缺点、综合评价、建议职级 |
| **三面** | HR面 | 人力资源面试评价 | 价值观：谦卑、渴求、聪慧 |
| **四面** | 终面（部门负责人/分管领导） | 校招综合评价表（技术类/综合类） | 潜力维度：系统思考、洞察力、追求极致、学习能力 / 创新能力、关注细节 |

> 另有"校招面试评分表"（综合类/技术类）用于校招终面，维度略有不同。

---

## 工作流

```
① 读取三件输入     JD(硬卡项) + 评估要点(真实看重什么/雷区) + 候选人简历(薄弱点)
    ↓
② 映射评分表维度   每轮面试 → 对应评分表的维度 → 结合岗位确定考察重点
    ↓
③ 生成定制问题     行为面试题（讲一个你过去...）+ 情境面试题（如果遇到...你会）
    ↓
④ 针对简历薄弱点追问   简历含糊处 → 设计追问（不放过"精通XX但描述含糊"的候选）
    ↓
⑤ 产出面试指南     4轮各自的考察重点+问题清单，落盘到岗位文件夹
```

## 阶段1：读取三件输入

### 输入1：JD 硬卡项
- 来源：`<岗位文件夹>/jd-optimized.md`（jd-craft 产出）或 `jd.md`
- 提取：硬卡项（必须考察的）、岗位职责（确定考察场景）

### 输入2：评估要点
- 来源：`<岗位文件夹>/评估要点.md`（analyze-resumes 产出）
- 提取：
  - **真实看重什么** → 面试核心考察方向
  - **硬卡项** → 一面/二面必须验证的
  - **雷区** → 面试中要重点排除的

### 输入3：候选人简历（薄弱点）
- 来源：归档简历 PDF（`data/.../已收集简历/`）或飞书招聘附件
- 提取：
  - **含糊处**（"精通Spine"但项目描述没有Spine细节 → 追问）
  - **时间空窗**（要问）
  - **量化缺失**（"负责XX"没数据 → 追问产出）
  - **夸大嫌疑**（职责超出岗位职级 → 验证）

## 阶段2：映射评分表维度

按轮次映射（详见 `references/scoring-rubrics.md`）：

### 一面（技术面）— 部门初试评价表
| 评分维度 | 考察什么 | 问题方向 |
|---------|---------|---------|
| 教育背景及专业培训 | 基础是否扎实 | 针对岗位要求的专业背景验证 |
| 原有经验应用到新工作 | 经验迁移能力 | "你过去做的XX，怎么应用到我们这个岗位？" |
| 专业问题/案例回答 | 真实技术水平 | 给具体技术案例让候选人拆解 |
| 专业工具应用 | 工具掌握深度 | "你用过XX做过什么？遇到什么问题？" |
| 弥补欠缺经验需要时间 | 学习曲线评估 | "你觉得这个岗位最大的挑战是什么？需要多久上手？" |

### 二面（业务负责人面）— 部门复试评价表
| 评分维度 | 考察什么 | 问题方向 |
|---------|---------|---------|
| 优点 | 核心优势 | 综合一面反馈 + 深挖亮点 |
| 缺点 | 核心短板 | 综合一面反馈 + 验证担忧 |
| 综合评价 | 整体判断 | 团队适配度 + 成长性 |
| 建议职级 | 定级 | 对标内部同岗位人员 |
| 结论 | 录用决策 | 录用/不录用/待定 |

### 三面（HR面）— 人力资源面试评价
| 评分维度 | 考察什么 | 问题方向 |
|---------|---------|---------|
| **谦卑** | 能否承认不足、接受反馈 | "讲一个你犯过的最大错误，怎么处理的？" |
| **渴求** | 对成长/成就的驱动力 | "你未来3年的目标是什么？在做什么准备？" |
| **聪慧** | 思维敏捷度、学习力 | 现场给一个新问题看反应（不追求答对，看思路） |

### 四面（终面）— 校招综合评价表（技术类/综合类）
| 评分维度 | 考察什么 | 问题方向 |
|---------|---------|---------|
| 系统思考（技术类） | 能否看全局、做架构判断 | "如果让你从0设计XX系统，你怎么拆？" |
| 洞察能力 | 能否看到本质 | "你觉得XX行业/技术未来3年会怎么变？" |
| 追求极致（技术类）/ 创新能力（综合类） | 有没有高标准/创新意识 | "你做过的最满意的作品/项目，为什么？还能更好吗？" |
| 学习能力 | 新事物吸收速度 | "最近学了什么新技术/新工具？怎么学的？" |
| 关注细节（综合类） | 细节敏感度 | 给一个有细节坑的案例，看能否发现 |

## 阶段3：生成定制问题

### 行为面试题（讲一个你过去…）
基于**简历里的具体经历**设计，不是通用题库：

```
❌ 废话题："请介绍一下你的项目经历"
✅ 定制题："你简历里写'负责BuuPo的AI调用链路'，能具体讲讲：
   1、这个链路设计了哪几个环节？
   2、Prompt管理你们怎么做的？版本控制吗？
   3、延迟控制你们的目标是多少？怎么实现的？
   4、遇到最大的坑是什么？怎么解决的？"
```

**设计原则**（详见 `references/question-templates.md`）：
- **锚定简历原文**：问题必须引用候选人简历里的具体表述
- **追问到细节**：技术题追问到"指标/数据/方案选型"，不能停在"你做过吗"
- **STAR 结构**：行为题用 Situation-Task-Action-Result 引导

### 情境面试题（如果遇到…你会…）
基于**岗位真实工作场景**设计：

```
情境题（产品岗）：
"如果我们要做一个AI创作工具的MVP，但研发只有2个人、3个月时间，
你会怎么定义MVP的范围？砍掉什么、保留什么？为什么？"

情境题（技术岗）：
"线上突然出现Spine动画内存泄漏，帧率掉到15fps，你排查的思路是什么？
你会先看什么？怎么定位是哪个动画的问题？"
```

## 阶段4：针对简历薄弱点的追问

这是本 skill 的**差异化价值**——不是出通用题库，而是**针对这个候选人简历的含糊处定制追问**。

### 追问触发器

| 简历信号 | 追问方向 | 示例 |
|---------|---------|------|
| "精通/熟练 XX"但项目无细节 | 要具体案例 | "你说精通Spine，做过最复杂的骨骼绑定是什么？多少骨骼？" |
| 时间空窗（≥6个月） | 问空窗原因 | "XX年X月到X月这段空窗是在做什么？" |
| "负责XX"无量化产出 | 要数据 | "你负责的XX模块，最终的用户量/留存/性能指标是多少？" |
| 职责超出职级（如"主导"但年限短） | 验证真实角色 | "你说的'主导'，具体是你做决策还是执行别人的决策？" |
| 频繁跳槽（<2年/次） | 问离职原因 | 看是追求成长还是逃避问题 |
| 技术栈跨度异常大 | 验证深度 | "你列了Unity/Unreal/Cocos，哪个是你真正深入的？" |

## 不做的事（显式边界）

- ❌ **不出通用题库**——所有问题必须锚定具体候选人的简历 + 具体岗位的 JD/评估要点
- ❌ **不替面试官做录用决策**——只出考察问题，决策权在面试官
- ❌ **不生成面评**——面评是面试官写的，本skill只出"考前准备"
- ❌ **不覆盖 JD 硬卡项的判断**——硬卡项由 analyze-resumes 初筛，本skill在面试环节验证
