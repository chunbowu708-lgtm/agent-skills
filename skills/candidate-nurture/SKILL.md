# 候选人保温与面评催收

## 这个 skill 解决什么

招聘的本质是"把人留住"，不只是"把人招来"。从初筛到入职中间几周，候选人处于"静默"——**静默=流失**（竞品挖角、候选人反悔、感受不到重视）。

但你现在缺的是：**有了预警数据（谁停滞了、谁欠面评了）之后，缺少配套的"怎么碰"**。`_daily_review.py` 已经算出了谁该跟进，本 skill 补上"今天该碰谁 + 用什么话术碰"。

## 与 recruit-followup 的边界

| 能力 | 归属 | 说明 |
|------|------|------|
| 算预警数据（停滞/欠收/待推进） | **recruit-followup**（`_daily_review.py`） | 本skill**只读**其产出 `_daily_report.json` |
| 产出"今天该碰谁+话术" | **本skill** | 读报告 → 匹配话术 → 行动清单 |
| 发消息给候选人 | **人工** | 本skill只给话术，不自动发 |

**前置条件**：跑本 skill 前必须先跑 `python notes/_daily_review.py`（不写表）生成 `_daily_report.json`。报告不存在时提示用户先跑对账。

## 工作流

```
① 读预警数据     读 _daily_report.json 的 stuck / feedback_overdue / to_advance
    ↓
② 分类+排优先级   按"流失风险"排序（面评欠收>停滞N天>待推进）
    ↓
③ 匹配话术       每个候选人 → 按"阶段+停滞原因"匹配保温话术模板
    ↓
④ 产出行动清单   "今天该碰谁 + 怎么碰"（话术+渠道建议），用户确认后执行
```

## 阶段1：读预警数据

```bash
# 前置：先跑对账（如报告不存在）
python notes/_daily_review.py

# 读报告
python -c "
import json
d = json.load(open('notes/_daily_report.json', encoding='utf-8'))
s = d.get('structured', d.get('battle_list', {}))
for k in ['feedback_overdue','stuck','to_advance']:
    print(f'{k}: {len(s.get(k,[]))}条')
"
```

**三类预警数据**（`_daily_report.json` → `structured`）：

| 数据 | 含义 | 流失风险 |
|------|------|---------|
| `feedback_overdue` | 面试已过但面试官没交面评 | 🔴 高（候选人等结果，拖久=凉） |
| `stuck` (dwell≥2天) | 卡在某阶段无进展 | 🟠 中高（看卡几天，≥5天=高危） |
| `to_advance` | 面评通过但没推进下一轮 | 🟡 中（候选人觉得"面了没下文"） |

---

## 阶段2：分类 + 排优先级

按"流失风险"从高到低排序：

```
优先级1 🔴 面评欠收 ≥2天（feedback_overdue 且面试过了48h+）
         → 候选人在等结果，每多等一天流失概率飙升
         → 行动：催面试官交面评（不是碰候选人）

优先级2 🔴 停滞 ≥5天（stuck 且 dwell_days≥5）
         → 候选人已经"凉"了，必须主动碰
         → 行动：碰候选人（保温话术）

优先级3 🟠 停滞 2-4天（stuck 且 dwell_days 2-4）
         → 还在窗口期，轻量保温
         → 行动：碰候选人（轻量话术）

优先级4 🟡 待推进下一轮（to_advance）
         → 面评过了但流程没动，候选人可能觉得慢
         → 行动：推进流程 + 跟候选人说一声"通过了，在安排下一轮"
```

**输出排序清单**（给用户看）：
```
=== 今日保温/催收清单 ===
🔴 [催面评] 张三 一面过了3天没面评 → 催面试官张杰
🔴 [保温]   李四 卡在"等约面"7天 → 碰候选人
🟠 [保温]   王五 卡在"二面后"3天 → 轻量碰
🟡 [推进]   赵六 一面通过2天没推进 → 推进+告诉候选人
```

## 阶段3：匹配话术（详见 references/nurture-scripts.md）

按**阶段 × 停滞原因**匹配话术。核心原则：

1. **催面评**（对面试官）：礼貌但明确，点出"候选人还在等"，必要时引用面评要点（面评全文可取，刚解锁）
2. **碰候选人**（对候选人）：不让对方觉得"被冷落"，给个具体进展/预期，不要空问候

### 话术匹配逻辑

| 场景 | 话术方向 | 示例（完整版见references） |
|------|---------|------------------------|
| 催面评（对面试官） | 礼貌催+候选人等结果 | "X总，张三一面过去3天了，候选人那边在等结果，方便尽快写下面评吗？" |
| 停滞在"等约面"（对候选人） | 给预期+保温 | "XX你好，这边还在协调面试官时间，预计这周内安排，你那边最近方便面试的时间是？" |
| 停滞在"面试后等结果"（对候选人） | 主动同步进展 | "XX你好，一面反馈我们在汇总中，预计X号前给你答复，感谢耐心等待～" |
| 面评通过待推进（对候选人） | 好消息+预期管理 | "XX你好，一面通过了🎉，这边在安排下一轮，预计下周，保持联系～" |
| 长期停滞（≥7天，对候选人） | 重新激活 | "XX你好，之前聊的XX岗位还在推进中，你那边近期看机会的意愿有变化吗？" |

## 阶段4：产出行动清单

**输出格式**：
```
=== 今日保温/催收行动清单 ===
生成时间：2026-07-01 09:30
数据源：_daily_report.json（对账时间 2026-07-01 09:00）

🔴【催面评·1人】
  张三 - 一面(技术面) 6-28 过去3天无面评
  面试官：张杰
  → 话术：张杰，张三一面过去3天了，候选人那边在等结果，方便尽快写下评价吗？
  → 渠道：飞书IM私聊

🔴【保温·2人】
  李四 - 卡在"等约面" 7天
  → 话术：李四你好，这边还在协调面试官时间，预计本周内安排面试，你最近方便的时间段是？
  → 渠道：飞书IM/电话

  王五 - 卡在"二面后等结果" 3天
  → 话术：王五你好，二面反馈我们在汇总中，预计7-4前给你答复，感谢等待～
  → 渠道：飞书IM

🟡【推进·1人】
  赵六 - 一面通过2天未推进
  → 行动：去飞书招聘推进下一轮 + 跟候选人说"通过了，在安排下一轮"
```

**AI 行为指导**：
- 清单生成后**先给用户看**，用户确认后再逐条执行（不自动发消息）
- 催面评的话术要让用户确认（措辞涉及对面试官的态度，不能太硬）
- 长期停滞（≥7天）的候选人，**先问用户"这个还要不要继续跟"**，可能该终止了

---

## 不做的事（显式边界）

- ❌ **不自动发消息**——只给话术+渠道建议，人工发（措辞+时机需人把关）
- ❌ **不算预警数据**——读 `_daily_report.json`，不重新算（那是 recruit-followup 的活）
- ❌ **不替用户做终止决策**——长期停滞的，提示用户判断，不自动终止
- ❌ **不碰面评内容分析**——催面评只催"提交"，面评内容分析是 talent-review 的活
