# 简历分析提示词模板

对每份简历，按以下顺序和维度分析。**严格按顺序跑，前面的维度可以提前终止分析。**

---

## 输入
- 岗位名 + 团队
- 该岗位的"评估要点.md"内容（真实看重什么/硬卡项/可放宽项/雷区/秒推信号）
- 候选人简历文本（已提取）

## 输出（每份简历）
```json
{
  "姓名": "...",
  "判定": "强推|可推|待定|不推",
  "一句话理由": "...",
  "判据": "...（待定/不推必填，强推/可推可省）",
  "推荐理由": "...（仅强推/可推）",
  "风险提示": "...（仅强推/可推，无则省）",
  "建议面试重点": "...（仅强推/可推）"
}
```

---

## 维度1：方向硬匹配（出局机制，最先跑）

候选人实际方向 vs 岗位方向是否对口？
- 读沉默信号：简历通篇没提岗位核心技术栈、只写无关技能 → 方向不对口的高概率信号
- **方向严重不对口 → 直接判"不推"，不再往下评，判据写明方向不匹配的具体表现**

## 维度2：硬性项达标（对照评估要点的"硬卡项"）

逐条对照评估要点的硬卡项（学历、经验年限、特定技能等）：
- 全部达标 → 继续
- 有不达标项，但属于评估要点的"可放宽项" → 标"这项不够但可能可破"，**倾向落"待定"而非"不推"**
- 有不达标项且非可放宽 → 落"不推"或"待定"（视严重程度）

## 维度3：项目含金量（核心区分项，决定强推 vs 可推）

看四点：
- **量化结果**：有 DAU/收入/性能提升等数字 → 项目有真实分量
- **角色深度**："负责/主导" vs "参与/协助" → 主力还是挂名
- **项目规模/知名度**：行业头部、大体量 → 加分；全是没听过的小项目 → 存疑
- **水分警惕**：自我标榜"核心成员"但描述空洞 → 标黄，不能轻信

读沉默信号：几年经验却只有学校项目、或项目全是"参与"没有成果 → 能力存疑 → 不轻易给强推。

## 维度4：稳定性与风险（雷区扫描）

读评估要点的"雷区"，逐条扫：跳槽频率、经历断档、跨行/跨方向跨度、短期经历扎堆。
- **出现雷区信号 → 不直接判死，但必须标出**，影响档位（强推降可推、可推降待定）

## 维度5：加分亮点（捞亮点）

读评估要点的"秒推信号"和"加分项"：
- 命中"秒推信号" → 直接托到"强推"
- 命中"加分项" → 同档内往上排

---

## 判定落档规则（偏宽松，怕漏人）

| 情况 | 落档 |
|------|------|
| 方向严重不对口 | 🔴不推 |
| 硬卡项不达标且非可放宽 | 🔴不推 |
| 硬卡项有不达标但属可放宽 | ⚪待定 |
| 方向对口+硬性达标+项目含金量足+无雷区 | 🟢强推 |
| 方向对口+硬性达标但项目偏弱或有风险 | 🟡可推 |
| 拿不准（信息不足/边界case） | ⚪待定（标"建议人工复核"） |

**核心：拿不准优先落"待定"+标"这里看不准建议人工复核"，不要自信地瞎判。"不推"只在有明确硬伤时用。**

---

## 反模式（绝对不要）
- ❌ 用百分比/权重评分（"匹配度78%"是错觉）
- ❌ 关键词匹配当判断（"JD有Unity简历有Unity=匹配"是幻觉）
- ❌ 把简历自我标榜当真（"核心成员"可能挂名）
- ❌ 维度平铺、硬伤被软性加分稀释（方向不对口必须一票否决）
- ❌ 把JD当金科玉律（JD是合同价，读可放宽项）
- ❌ 碰薪资（BOSS收简历时已按薪酬期望筛过）
