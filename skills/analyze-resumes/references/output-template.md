# 产出格式模板

评估结果输出到 `notes/简历评估_{M.DD}.md`（NOTES_DIR 留空则用项目根 notes/），同时在终端展示。整个文件分三部分。

---

## 第一层：自筛（给 HR）

### 结构

```markdown
# 简历评估 · {M.DD}

## {岗位名}（{团队}） — {N} 份

| 姓名 | 判定 | 一句话理由 |
|------|------|-----------|
| {姓名} | {判定emoji+档位} | {一句话} |
| ... | ... | ... |

### 待定/不推 判据（供你复核）
- **{姓名}**（{档位}）：{判据}。{可选：上调建议}
- ...
```

### 判定 emoji

| 档位 | emoji | 含义 |
|------|-------|------|
| 强推 | 🟢 | 亮点突出、方向正、硬性项达标，优先推给业务 |
| 可推 | 🟡 | 方向对口、基本达标，值得安排面试 |
| 待定 | ⚪ | 有不确定区（拿不准），需 HR 复核再定 |
| 不推 | 🔴 | 有明确硬伤（方向不对口等），不建议推 |

### 排序规则
- 组内按 强推 > 可推 > 待定 > 不推 排序
- 同档内：含金量高的在前
- 强推/可推只给一句话理由（细节在第二层）
- 待定/不推 必须单独附判据（失败反哺入口）

---

## 第二层：推荐摘要（给业务部门）

只对"强推/可推"生成。每人一段，可直接复制发给面试官。

### 结构

```markdown
## 推荐摘要（可发给业务）

### {姓名} · {岗位名}
**推荐理由**：{2-3句，核心亮点：经验年限、相关项目角色、量化产出、技术栈对口度}

**风险提示**：{1句，诚实标注：跳槽频率、短经历、角色存疑等。没有就不写这整段}

**建议面试重点**：{1-2句，把"简历解不开的疑问"转化为"面试该问什么"}
```

### 要点
- 三段式：推荐理由 + 风险提示 + 建议面试重点
- 不是简历复述，是给面试官的决策加速
- 风险提示诚实：有就写，没有整段省略
- 不带薪资、不带隐私（电话邮箱等不出现）

---

## 兜底区：需人工看原文件

文本提取异常的简历单列一区，不混进判定：

```markdown
## 需人工看原文件
- **{姓名}**：{issue}，路径：{文件路径}
```

---

## 第三层：按档位分发简历文件（评估后必做）

评估 md 写完后，把当天归档的简历**按判定档位移入日期目录下的子文件夹**，让用户能直接打包"强推/"或"可推/"发给业务部门。

### 目录形态

```
{岗位}/已收集简历/{M.DD}_N份/
├── 强推/           ← {姓名}_{岗位简称}_{届}.pdf ...
├── 可推/
└── 待定·不推/      ← 待定和不推都放这里；兜底区文本异常的也放这，标注"需人工看原件"
```

### 操作要点
- 每份简历 `mv`（不复制）到对应档位文件夹
- 兜底区文本异常的简历 → "待定·不推/"，文件名加 `[需人工看原件]` 前缀
- 分档后跑一次 `verify_archive.py` 确认份数没丢（闸门已兼容档位子目录，会递归计数）

### 终端展示
```
已按档位分发到 {岗位}/已收集简历/{M.DD}_N份/：
  强推/ {N}人  → 可直接打包发给业务
  可推/ {N}人
  待定·不推/ {N}人
```
