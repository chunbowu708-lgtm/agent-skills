# 需求澄清问题库

> 阶段2 产出"需求澄清.md"时，按岗位类型选问题。这些问题是把 HR "和业务对需求"的经验固化的关键。

## 通用问题（所有岗位都要问）

### 岗位定位
- 这个岗是**替换**某人（谁？为什么离？）还是**新增**编制？
- 团队现在**最缺什么样的人**？（补技术短板？扩产能？带新人？）
- 这个岗**最不能妥协的 3 项**能力/经验是什么？（按优先级排，第1项是硬卡）

### 团队现状
- 团队多大？直接汇报给谁？汇报对象的职级和风格？
- 团队的技术栈/工具链/工作方式（敏捷？瀑布？）
- 团队目前最大的挑战是什么？（这个岗来了主要解决什么）

### 薪资与级别
- **薪资范围**？（必问。候选人最关心，JD不写会被问爆）
- 对应职级？（初级/中级/资深/专家，公司职级体系）
- 有期权/股票吗？

### 急迫度
- 期望什么时候到岗？
- 有备选候选人了吗？（影响是否要快速推进）
- 接受远程/混合办公吗？

## 按岗位类型的专项问题

### 技术岗（开发/测试/运维）

- **技术栈硬性项**：必须会哪些语言/框架/工具？（按"必须/优先"分）
- **项目经验**：必须做过什么类型的项目？（上线项目？高并发？特定业务领域？）
- **代码/架构能力**：看重"能写"还是"能设计架构"？带不带人？
- **AI 工具使用**：对 Cursor/Copilot/Claude Code 的使用有要求吗？（新趋势，部分团队硬性要求）
- **坑点确认**：JD 里的"熟悉 XX"到底是"听过"还是"做过项目"？

### 设计岗（UI/交互/美术/动效）

- **作品集方向**：必须有什么类型的作品？（写实/Q版/二次元？移动端/PC？）
- **工具硬性要求**：Figma？Spine？Maya？Photoshop？版本有要求吗？
- **风格匹配**：团队现有美术风格是什么？新人必须风格一致还是带来新风格？
- **协作方式**：和策划/程序的协作流程？需要出多少版本？修改频率？
- **坑点确认**：作品集要求是"几张图"还是"完整项目case"？

### 运营岗（内容/用户/社区/活动）

- **必须的渠道经验**：做过哪些平台？（Discord/Reddit/B站/抖音？国内还是海外？）
- **数据指标要求**：背过什么KPI？（DAU/留存/内容产出量/创作者活跃度？）
- **内容判断力**：怎么考核"内容审美"？（给案例让候选人分析？）
- **创作者/用户关系**：需要直接和创作者打交道吗？有运营过KOL/创作者社群吗？
- **坑点确认**：JD 里的"UGC经验"是指"做过UGC产品"还是"在UGC平台运营过内容"？

### 产品岗（产品经理/数据PM）

- **产品类型**：必须负责过什么类型的产品？（C端/B端？工具/平台/社区？）
- **海外/英文**：海外岗必须英文可工作吗？需要对接海外团队吗？
- **独立负责**：要"独立负责一个模块"还是"支持某个产品线"？从0到1还是迭代？
- **数据能力**：要会SQL吗？要看仪表盘吗？要做过AB测试吗？
- **AI 方向**（AI产品岗特有）：对AI/LLM/AIGC的理解要求多深？用过哪些工具？
- **坑点确认**：JD 里的"产品经验"是"写过文档"还是"推进过项目落地"？

## 问题选择的判断逻辑

阶段2 产出澄清问题时，不要全量甩给业务（太多=业务懒得回）。按这个逻辑筛选：

1. **JD 体检里标❌的维度** → 对应的问题必问（缺失项精准提问）
2. **硬卡项模糊** → "最不能妥协的3项"必问（这是 analyze-resumes 评估的根基）
3. **薪资/急迫度** → 必问（影响寻访策略和推进节奏）
4. **岗位类型专项** → 选最可能踩坑的 2-3 个问（不是全部问一遍）

**输出原则**：问题**不超过 12 个**，分 4 类（定位/团队/薪资级别/专项），让业务 5 分钟能答完。
