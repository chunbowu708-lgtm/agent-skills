# 归档命名规则

## 目录结构

```
{归档根目录}/                              ← $ARCHIVE_ROOT
├── {项目团队}/                            ← 如"项目A团队"、"项目B团队"
│   ├── {岗位名}/已收集简历/               ← 无中间团队层
│   │   └── {M.DD}_{N}份/
│   └── {团队名}/{岗位名}/已收集简历/      ← 有中间团队层
│       └── {M.DD}_{N}份/
```

归档时**先 ls 确认实际路径再操作**。不同项目层级不同，猜错路径比多花2秒查一次严重得多。
已收集简历文件夹名统一为"已收集简历"（历史遗留的其他命名不改）。

## 文件命名

### 普通岗（只有简历 PDF）
- 正职：`{姓名}_{岗位简称}_{经验}年.pdf`
- 实习：`{届}_{学历}_{姓名}_{学校}.pdf`
- 岗位简称用本地文件夹名
- 简历文件名不得含薪资信息（薪资脱敏）

### 美术岗（简历 + 作品）
- 候选人自己发的现成 zip 且命名规范 → 直接用
- 候选人发的 zip 命名不规范（含电话、原始前缀等）→ 重新命名后归档
- 简历和作品分开发 → 打包前可在日期文件夹内建临时子文件夹暂存，打包完成后删除子文件夹
- 打包后：`{姓名}_{岗位简称}_{经验}年_简历加作品.zip`

### 美术岗（只有简历，作品已含在 PDF 中）
- 和普通岗一样

## 归档后更新文件夹命名

```bash
COUNT=$(find "$DIR" -maxdepth 1 -type f | wc -l)
mv "$DIR" "$(dirname "$DIR")/6.10_${COUNT}份/"
```
用 `find -maxdepth 1 -type f` 而不是 `ls`，因为临时子文件夹（打包暂存）会导致 `ls` 虚报份数。
