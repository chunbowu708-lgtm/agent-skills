# 候选人人才画像（匹配覆盖图）

## 这个 skill 解决什么

HR 筛人的真实场景不是"给每个候选人打分"，而是**"同岗位 N 个候选人摆一起，看谁缺哪块"**——找短板，不是找总分。

本 skill 把简历 + JD 要求项转成**匹配覆盖图**：
- **不打分**（"沟通能力 8/10"是假精确，AI 负不了这个责）
- **只做覆盖判断**：✅满足 / ⚠️部分 / ❌缺失（这个判断 AI 能负责）
- **横向对比**：同岗位多人并排，谁缺哪块一眼看出

## 设计决策：为什么用覆盖图而非雷达图

| 方案 | 问题 |
|------|------|
| ❌ 雷达图（打分） | 强迫给每个维度一个数值，而数值不可靠（"创新能力 7/10"依据是什么？） |
| ✅ 覆盖图（不打分） | 只做"有/无/部分"的事实判断，绕开打分，信息更诚实 |

覆盖图本质是**热力表**：行是候选人，列是 JD 要求项，单元格是 ✅/⚠️/❌。

---

## 工作流

```
① 读取输入     评估报告(候选人判定+理由) + JD(要求项)
    ↓
② 提取要求项   从JD拆出N个可验证的要求项（硬卡项+优先项）
    ↓
③ 逐人覆盖判断 每个候选人 × 每个要求项 → ✅/⚠️/❌
    ↓
④ 生成覆盖图   HTML表格（行=候选人，列=要求项），高亮风险点
    ↓
⑤ 落盘+展示    HTML存岗位文件夹，浏览器打开看
```

## 阶段1：读取输入

### 输入1：评估报告
- 来源：`notes/简历评估_M.DD.md`（analyze-resumes 产出）
- 提取：每个候选人的**判定**（强推/可推/待定/不推）+ **推荐理由** + **风险提示**
- 只取强推/可推（待定/不推不画画像）

### 输入2：JD 要求项
- 来源：`<岗位文件夹>/jd-optimized.md` 或 `评估要点.md`
- 提取：拆成 6-10 个**可验证的要求项**
  - 硬卡项：如"3年Unity经验""独立负责过模块""英文可工作"
  - 优先项：如"有UGC经验""做过AIGC""游戏行业背景"

**要求项的标准**：必须是能从简历判断"有/无/部分"的，不能是"责任心强"这种不可验证的软素质。

## 阶段2：生成覆盖图

### 覆盖判断规则

| 标记 | 含义 | 判断依据 |
|------|------|---------|
| ✅ | 满足 | 简历有明确证据（具体项目/年限/数据） |
| ⚠️ | 部分/模糊 | 有相关信号但不够硬（"接触过"但没项目，或有年限但无产出数据） |
| ❌ | 缺失 | 简历无任何相关信号 |
| — | 不适用 | 该要求项对此候选人不适用（如跨行转岗的"游戏经验"） |

### HTML 输出格式

用 `scripts/generate_profile.py` 生成独立 HTML（照 storage-analyzer 模式，离线可看）。

**HTML 结构**：
```html
<h1>匹配覆盖图 · <岗位名></h1>
<table>
  <tr>
    <th>候选人</th><th>判定</th><th>要求项1</th><th>要求项2</th>...<th>风险点</th>
  </tr>
  <tr>
    <td>张三</td><td>🟢强推</td><td>✅</td><td>⚠️</td>...<td>Spine经验描述含糊</td>
  </tr>
  ...
</table>
```

**视觉规则**：
- ✅ 绿色背景 / ⚠️ 黄色背景 / ❌ 红色背景（热力图效果）
- "风险点"列汇总该候选人的主要短板（从评估报告的风险提示提取）
- 按判定排序：强推 → 可推（待定/不推不显示）

## 阶段3：使用场景

### 场景1：筛人决策（发给业务前）
- 看覆盖图：哪个候选人"红格最少"（短板最少）→ 优先推进
- 看风险点列：哪个候选人的短板是硬伤（❌在硬卡项）→ 暂缓或放弃

### 场景2：面试准备（配合 interview-guide）
- 覆盖图的 ⚠️ 和 ❌ 格 → 就是面试要追问的点
- 直接把覆盖图丢给 interview-guide："⚠️的这几项重点设计追问"

### 场景3：业务汇报（给业务看）
- 覆盖图比"评估报告全文"更直观——业务一眼看出"这批人整体缺什么"
- 帮助业务校准 JD："你看这批候选人普遍缺UGC经验，是不是JD要调整或者放宽？"

## 脚本

### `scripts/generate_profile.py`

输入：评估报告路径 + 岗位文件夹路径
输出：`<岗位文件夹>/talent-profile.html`

```bash
python "C:/Users/wuchunbo/.agents/skills/talent-profile/scripts/generate_profile.py" \
  --report "notes/简历评估_7.01.md" \
  --position "data/在招岗位候选人管理/.../岗位文件夹"
```

脚本逻辑：
1. 解析评估报告（markdown 表格 → 候选人数据）
2. 读 JD 要求项（jd-optimized.md 的硬卡项/优先项）
3. AI 逐人逐项判断覆盖度（这部分由 AI 在对话中完成，脚本负责生成 HTML 壳）
4. 输出 HTML（含样式，离线可看）

> ⚠️ 覆盖判断（✅/⚠️/❌）是判断活，由 AI 在对话中完成，不写死规则。脚本只负责把 AI 判断的结果渲染成 HTML。

---

## 不做的事（显式边界）

- ❌ **不打分**——只做 ✅/⚠️/❌ 覆盖判断，不输出"8/10"这种假精确数值
- ❌ **不重新评估简历**——读 analyze-resumes 的产出，不重新判断强推/可推
- ❌ **不生成面试问题**——那是 interview-guide 的活（但覆盖图的⚠️❌可作为它的输入）
- ❌ **不覆盖评估报告**——HTML 是评估报告的可视化补充，不替代
